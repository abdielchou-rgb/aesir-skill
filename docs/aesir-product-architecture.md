## 一、产品定位

### 一句话描述

> **你的私人故事引擎——手机随时捕捉灵感，帮你从碎片长成故事。**

### 目标用户

| 用户层 | 使用场景 | 付费意愿 | 传播力 |
|--------|---------|---------|--------|
| **轻度装逼用户** | 发小红书/抖音/微博的故事碎片、心情分享 | 低 | 极高 |
| **内容创作者** | 用碎片做营销策划、AI 短视频、漫画剧本 | 中 | 中 |
| **严肃创作者** | 网文/剧本/漫画长篇写作 | 高 | 低 |

### 核心价值

- **零门槛**：不用配 API key，不用装 Python，打开就能用
- **越用越懂你**：蒸馏别人的作品，形成专属于你的原子库和收敛逻辑
- **无缝升级**：手机碎片 → 桌面长篇，同一个风格档案贯穿始终

## 三、用户旅程

### 场景 A：手机端碎片捕捉（全用户）

```mermaid
用户掏出手机 → 打开 Æsir PWA
     ↓
输入一句话："今天地铁上看到一个女孩在哭"
     ↓
Æsir 对话（前3次驯化）：
  "这个场景最触动你的是她本人，还是周围人的反应？"
  "你想用这个场景表达什么情绪？"
     ↓
发散 5 条世界线（演示模式或 AI 模式）
  1. 温暖治愈 → 陌生人的善意
  2. 悬疑暗流 → 她为什么哭
  3. 诗意思辨 → 城市里哭泣的陌生人
  ...
     ↓
用户选一条 → 加约束 → 生成 300 字故事片段
     ↓
保存 / 复制到小红书 / 分享到朋友圈
```

### 场景 B：桌面端长篇延伸（付费用户）

```mermaid
用户在手机上保存了 20+ 片段
     ↓
坐到电脑前，打开 Æsir 网页版 / 桌面版
     ↓
查看"故事档案馆" → 选中一个感兴趣的片段
     ↓
"我想把这一段扩展成一个完整的故事"
     ↓
选择弧线 → Novel OS 接管 → 生成章节大纲
     ↓
逐章写作 + 审计反馈 + 修订
     ↓
导出为 Markdown / Epub
```

### 场景 C：蒸馏个人原子库（付费用户）

```mermaid
用户说："帮我分析这三本村上春树的小说"
     ↓
上传 .epub / .txt 或粘贴文本
     ↓
蒸馏管道：
  1. 表层：句长分布、对话比例、感官通道
  2. 中层：情节结构标注、情感弧线
  3. 原子提取：匹配 12 爽点 + 6 调性 + 6 冲突
     ↓
生成"村上风格"蒸馏包
     ↓
下次发散时，系统说：
"根据你的村上风格蒸馏包，这条世界线特别适配"
```

## 四、技术升级方案

### 4.1 统一 `aesir-writing` 核心包

**当前问题**：`aesir-mcp-server.py` 和 `aesir-writing/` 是两套代码，各自操作不同的存储。

**升级方案**：废弃 `aesir-mcp-server.py`，全部能力合并到 `aesir-writing` 包。

```
aesir-writing/
├── __init__.py          # 版本号
├── main.py              # CLI 入口（改写）
├── phone/               # 新增：手机交互层
│   ├── app.py           # PWA 后端 + 演示模式引擎
│   └── demo_engine.py   # 演示模式发散收敛（无 API key 可用）
├── core/                # 核心能力（保留并扩展）
│   ├── atoms.py         # 原子库（从 aesir-mcp-server 移入）
│   ├── diverge.py       # 发散引擎
│   ├── converge.py      # 收敛引擎
│   ├── engine.py        # 探索-利用平衡 + 原子兼容性
│   └── intent.py        # 意图卡片
├── distill/             # 蒸馏管道
│   ├── surface.py       # 表层 stylometry
│   ├── structural.py    # 中层 LLM
│   ├── atom_extract.py  # 原子提取
│   └── profile.py       # 风格档案管理器
├── novel/               # Novel OS 模块（原 aesir-writing/ + novel-os）
│   ├── story.py         # StoryModel
│   ├── orchestrator.py  # 编排引擎

│   └── arcs.py          # 弧线跟踪

│   ├── web.py           # FastAPI web 服务
│   └── sync.py          # 同步服务
└── knowledge/           # 知识底座引用
    └── base.py          # 加载 knowledge-base.md
```

### 4.2 演示模式引擎（无 API key 可用的核心）

这是 Tier 0 免费传播的关键——用户不需要配置任何东西，打开就能用。

```
demo_engine.py 包含：
  - 发散：30 前提模板 × 24 钩子模板 × 6 调性 × 6 冲突 × 6 主角（组合空间：30×24×6×6×6）
  - 收敛：6 调性 × 4 故事片段变体
  - 模板选择逻辑：content_hash(用户输入关键词 + 时间种子)
  - 目标：让用户感觉"它真的在听我说"，而不是"它在背模板"
  - 上限：100+ 次使用不会出现重复组合
```

### 4.3 Tier 分层实现

| 能力 | Tier 0 (免费) | Tier 1 (免费) | Tier 2 (付费) |
|------|-------------|-------------|-------------|
| 发散/收敛 | 演示模式 | 全 AI 模式 | 全 AI 模式 + 云端 |
| API key | 不需要 | 用户自配 | 系统提供 |
| 蒸馏 | 表层 3次/月 | 全层 30次/月 | 无限 |
| 风格档案 | 本设备 | 本设备 | 云端同步 |
| 原子库 | 预设 30 | 预设+蒸馏 | 预设+蒸馏+市场 |
| 长篇小说 | - | 基础 | 完整 |
| 导出 | 文本复制 | Markdown | 全格式 |
| 价格 | ¥0 | ¥0 | ¥39/月 |

## 七、核心原则

1. **Tier 0 不需要任何配置**——打开网页就能用。这是传播的根基
2. **演示模式不叫"演示模式"**——用户不需要知道这是否用了 AI。就叫"故事织机"
3. **免费用户也能产生高质量内容**——只是量少、不能导出、不能云同步
4. **风格档案是真正的锁**——用户用得越久，越离不开自己的风格档案
5. **蒸馏包市场是增长引擎**——用户创造的风格档案可以分享和出售
