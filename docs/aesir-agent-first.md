# Æsir agent-first 架构方案

> PWA 是入口，不是产品。Agent 是产品。

---

## 一、核心假设

**用户不需要另一个 app。用户已经有 agent 了。**

| 现有 agent 平台 | 月活 | Ësir 的形态 |
|---------------|------|-----------|
| Claude Desktop / App | 数千万 | MCP Server（已实现，38 个工具） |
| Coze | 数千万 | Bot 配置（已实现，4 个工作流） |
| ChatGPT / GPTs | 数亿 | GPTs Action（待实现） |
| 微信 / 飞书 / 钉钉 | 数十亿 | 机器人接入（待实现） |
| Cursor / Windsurf | 数百万 | MCP Server（已兼容） |

**策略：让用户的 agent 学会讲故事，而不是让用户学用 Æsir。**

---

## 二、产品形态分层

```
                   用户与 agent 的日常对话流
                             │
                ┌────────────┴────────────┐
                │                         │
        ┌───────▼───────┐        ┌───────▼───────┐
        │  agent 原生对话  │        │  Æsir Skill   │
        │  (用户自己的 LLM) │◄──────►│  (MCP服务)     │
        └───────────────┘        └───────┬───────┘
                                         │
                               ┌─────────▼─────────┐
                               │    Æsir Engine     │
                               │  (本地/云端)        │
                               │  原子库 蒸馏 档案    │
                               └───────────────────┘
```

**用户看到的：** 她的 agent 突然能帮她构思故事、分析文本、发散去重。  
**背后：** agent 调用了 Æsir 的 MCP 工具——工具返回结构化数据，agent 用自己的语言能力生成最终文本。

## 三、三个入口，一个引擎

### 入口 1：MCP Server（主力）

用户把 Æsir 的 MCP Server 配置到她的 agent 中。之后她跟 agent 说"帮我想个故事"，agent 自动调用 `diverge` → 展示世界线 → 用户选择 → 调用 `converge` → 生成片段。

**覆盖平台：** Claude Desktop/App、Cursor、Windsurf、任何支持 MCP 的客户端

**部署方式：**
```json
// Claude Desktop 配置中加一行
"mcpServers": {
  "aesir": {
    "command": "python3",
    "args": ["path/to/aesir-writing/server/mcp.py"]
  }
}
```

**用户体验：** 用户不感知 Æsir 的存在。她只是觉得自己的 agent 多了"写故事"的能力。

### 入口 2：Coze Bot（零部署）

用户直接在 Coze 市场搜索"Æsir"→ 安装 → 开始对话。

**覆盖平台：** Coze Web / App / 微信小程序 / 飞书

**用户体验：** 跟一个专门讲故事的 bot 对话。

### 入口 3：PWA 伴侣（轻量辅助）

PWA 不做推理，只做三件事：
1. **快速捕获**——手机锁屏时想起一个念头，点进去记一句话
2. **历史浏览**——浏览之前 agent 生成的所有片段
3. **社交分享**——把 agent 生成的片段生成卡片分享出去

**用户关系：** PWA 是 agent 的"存档和分享客户端"，不是产品本身。

```
                    ┌─────────────┐
                    │  用户&agent  │
                    │  对话流      │
                    └──────┬──────┘
                           │ 调用工具
                    ┌──────▼──────┐
                    │  Æsir MCP   │
                    │  38 个工具   │
                    └──────┬──────┘
                           │ 保存/读取
              ┌────────────┼────────────┐
              │            │            │
       ┌──────▼─────┐ ┌───▼───┐ ┌────▼────┐
       │ 本地存储    │ │ PWA   │ │ 云端     │
       │ ~/.aesir/  │ │ 历史   │ │ (付费)   │
       └────────────┘ │ 分享   │ └─────────┘
                      └───────┘
```

## 四、Agent 如何驱动故事创作

关键设计：**agent 负责对话和语言生成，Æsir 负责结构数据和工具逻辑。**

| 步骤 | Agent 做的事 | Æsir 做的事 |
|------|------------|------------|
| 1. 用户说想法 | 理解意图，判断是否前3次 | — |
| 2. 澄清对话 | 用自然语言提问 | 记录用户回答到 session |
| 3. 发散 | 向用展示世界线文本 | `diverge` 返回 5 条结构化配置 |
| 4. 用户选择 | 解析用户的选择 | `converge` 返回配置 + 约束历史 |
| 5. 讲故事片段 | **用 LLM 基于配置生成 500 字开场文本** | `save` 存档 + 未选入发酵池 |
| 6. 破坏约束版 | **用 LLM 生成相反方向版本** | `get_ferment_pool` 回顾 |
| 7. 分析文本 | — | `distill` 更新风格档案 |

**关键洞察：** agent 用自己的 LLM 能力生成文本，Æsir 不参与文本生成。Æsir 只做 agent 自己做不了的事：原子管理、风格档案、发酵池、蒸馏管道。

## 五、PWA 改成"存档和分享客户端"

当前的 `aesir-phase1.html` 定位改为：**不生成内容，只展示 agent 生成的内容。**

```
PWA 功能精简为：
┌─────────────────────────┐
│  Æsir 故事簿             │
│                         │
│  ✎ 快速捕获（一句话）      │
│                         │
│  ── 最近生成 ──          │
│  📖 故事片段 1           │
│  📖 故事片段 2           │
│  📖 故事片段 3           │
│                         │
│  🌱 发酵池（3 条）        │
│  ◉ 我的风格档案           │
│  ✦ 生成分享卡片           │
└─────────────────────────┘
```

用户行为流：
1. 在 Claude/Coze 里跟 agent 对话，生成故事片段
2. 打开 PWA，看到最近生成的片段列表
3. 选中一个，编辑，生成分享卡片
4. 发到小红书/朋友圈

**Agent 生成的内容自动同步到 PWA，PWA 是 agent 的"历史记录和分享出口"。**

---

## 六、为什么这样做

**之前思路的问题：PWA 想自己做完一切——对话、发散、收敛、存储、分享。**
这要求 PWA 自己有 LLM 能力（需要用户配 API key）、有状态管理、有复杂的交互逻辑。做得越多，越重，越不像一个手机友好的产品。

**新思路：PWA 只做它擅长的——展示和分享。**
对话和生成交给 agent——agent 本来就在用户的对话流里，不需要用户"打开另一个 app"。

**类比：**

| 产品 | 核心能力 | 客户端 |
|------|---------|--------|
| ChatGPT | 对话 + 推理 | Web/App（展示历史） |
| Æsir | 故事工具 | Agent + PWA（展示和分享） |

ChatGPT 的 Web 端不做推理，推理在云端。你的 PWA 也不做推理，推理在 agent 里。
