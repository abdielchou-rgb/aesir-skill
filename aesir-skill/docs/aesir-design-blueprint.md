# Æsir · 故事织机 — 完整设计蓝图

---

## 一、设计目标

### 核心命题

> **让每个人都能用一个模糊的想法，随时随地生成有趣的故事，并且系统越用越懂你。**

### 五条目标

| # | 目标 | 对应用户诉求 | 非目标（明确不做） |
|---|------|------------|-------------------|
| 1 | **零门槛启动**：用户说一句话就能开始，不需要学习任何东西 | "我就是有个模糊的想法" | 不做长篇小说的章节管理 |
| 2 | **涌现优先**：系统不替代用户创作，而是用发散-收敛循环激发用户自己的创意 | "帮我打开思路，不是帮我写" | 不做"一键生成完本小说" |
| 3 | **越用越对味**：风格档案 + 行为蒸馏让系统在 3 次使用内明显"更懂你" | "它知道我喜欢什么样的故事" | 不做千人一面的通用生成 |
| 4 | **手机随时可用**：PWA 优先，5 秒内完成一次灵感记录 | "等咖啡的时候能用" | 不做桌面端先行的重应用 |
| 5 | **可分享、可导出**：每个故事片段和风格档案都是可传播的社交资产 | "我想让别人看到我的创意" | 不做封闭的内容花园 |

## 二、设计哲学

### 哲学一：约束产生涌现，工具不替代人

系统不"替用户写"，而是：
- 给出足够多的可能性（5-8 条世界线）让用户感到有选择空间
- 用约束而非填空来引导用户——"给我写个修仙故事"不是填空，是约束空间的定义
- 保留 30-50% 的"涌现间隙"——生成的片段不说完、不说满，留给用户自己去填充和修改
- 破坏约束版本（deliberate constraint break）作为每轮收敛的标配输出——提醒用户"结构之外还有可能性"

### 哲学二：三次驯化，从不空手而来

前三次使用强制走"对话优先"路径：
- 第一次：系统不生成世界线，只提问。"你想写什么样的故事？你最近看过什么喜欢的作品？"
- 第二次：基于第一次的回答生成少量（3 条）世界线，收集隐式反馈
- 第三次：开始全功能发散，但附带说明"这是根据你对 XX 的偏好调整过的"
- 三次之后：正常使用，同时开放蒸馏功能

设计意图：不让第一次用户面对 8 条世界线感到淹没。

### 哲学三：让用户积累数据资产，而非消费功能

用户离开系统的迁移成本不是"这个功能别处没有"，而是"我的风格档案在这里"。系统维度：
- 每一次选择都在塑造个人的叙事风格档案
- 每一次蒸馏都在创造可复用的风格资产
- 每一个存档的故事都是个人风格史的切片
- **风格档案可以导出，但导出后失去的是"下次生成的上下文关联"——这才是真正的锁定**

### 哲学四：原子由系统推导，不凭感觉预设

原子库的三层结构：
- 基础层（30-50 个跨文化基元）：来自叙事学经典框架（Propp、Campbell、Booker、Polti），系统预设
- 类型层（每类型 20-30 个特色原子）：由系统从同类型大量文本中蒸馏提取后内置
- 用户层（从用户个人语料和行为中提取）：通过蒸馏管道逐步构建

**原子库不是封闭的。用户可以蒸馏、可以组合、可以分享。系统不声称"原子只有这几种"，而是说"这是目前能看到的，你可以自己制造新的"。**

### 关键数据结构

**核心实体：**
- `Fragment`：创意碎片（用户自由输入的原始素材，归类但不锁定）
- `Worldline`：一条完整的世界线（前提句 + 故事钩子 + 关联原子标签 + 状态）
- `Atom`：叙事原子（id + 名称 + 描述 + 所属层级 + 所属类型）
- `StyleProfile`：用户风格档案（节奏指纹 + 视角偏好 + 原子分布 + 情感倾向）
- `Distillate`：蒸馏产物（源语料哈希 + 原子频次分布 + 节奏特征向量 + 元数据）

**原子之间的可能连接关系：**
- 原子 A → 原子 B（因果链）
- 原子 A + 原子 C（高频组合模式）
- 原子 A ── 类型 —— 原子 D（同类型变体）
- 原子 A ── 冲突 —— 原子 E（相反/互补关系）

## 四、工程蓝图

### 技术选型

| 层 | 技术 | 选择原因 |
|---|------|---------|
| 前端框架 | PWA (Lit / Svelte 轻量方案) | 手机优先，包体小，离线可用 |
| 后端核心 | Python (FastAPI) + MCP 协议 | agent 生态兼容性 + ML 生态 |
| 原子库 & 风格档案 | SQLite (本地优先) / 可选云端同步 (PostgreSQL) | 零配置启动，可选同步 |
| 蒸馏管道 | NLTK / spaCy (表层) + LLM API (中深层) | 工具链成熟 |
| 发酵池 | SQLite 中维护状态 + 定时唤起的轻量 scheduler | 不需要实时，离线也工作 |
| LLM 接入 | OpenAI / Claude / 本地模型 (Ollama) 多 provider | 用户可选择，降低依赖风险 |
| 文件处理 | Calibre (epub 解析) + python-magic (格式检测) | 支持用户上传蒸馏 |

### 数据落盘策略

```
用户设备本地 SQLite:
  └─ aesir.db
       ├─ atoms             原子库
       ├─ fragments         创意碎片
       ├─ worldlines        世界线
       ├─ sessions          交互会话（行为信号源）
       ├─ distillates       蒸馏产物
       ├─ style_profile     风格档案
       ├─ fermentation_pool 发酵池
       └─ archive           故事档案馆
可选择同步到云端（订阅用户）:
  └─ PostgreSQL (用户风格档案 + 档案馆 + 蒸馏包索引)
```

### 外部集成接点

| 集成类型 | 集成对象 | 集成方式 | 优先级 |
|---------|---------|---------|-------|
| **故事产出** | 轻量 Markdown 编辑器 (内置) | PWA 内嵌 | P0 |
| **导出** | Markdown / TXT (通用) | 浏览器下载 / 复制 | P0 |
| **导出** | StoryForge 格式 | export adapter | P1 |
| **导出** | ElyHa 格式 | export adapter | P2 |
| **导入蒸馏** | epub / txt / paste | 文件上传 / 粘贴 | P0 |
| **AI provider** | OpenAI / Claude / Ollama | 用户配置 API key | P0 |
| **分享** | 公开链接 | 后端生成短链 | P1 |

## 五、阶段性 MVP 计划

### 开发节奏总览

```
Phase 0: 原型验证（6 周）
Phase 1: MVP 上线（10 周）
Phase 2: 蒸馏与学习（14 周）
Phase 3: 商业闭环（18 周）
```

### Phase 0: 原型验证（6 周）

**目标：** 验证核心命题——"人机对话发散-收敛循环"是否能产出让用户觉得"有用"的故事方向。不做界面，不做 PWA，不做蒸馏。纯命令行 + LLM 调用。

| 周 | 内容 | 可交付 |
|----|------|--------|
| 1-2 | 预设原子库 V1（30 个跨文化基元），数据结构和原子组合器逻辑 | `atoms.json` + 组合器 Python 模块 |
| 3-4 | 基于原子组合 + LLM 调用的发散模块，输入"一句话模糊想法"→ 输出 5 条世界线 | CLI 可执行：`python diverge.py "一个关于错过的故事"` |
| 5-6 | 收敛模块 + 最基本的"对话意图识别"（不是 UI，是 prompt 解析用户想要的约束方向） | CLI 端到端：模糊想法 → 发散 → 人工约束文本输入 → 收敛 → 输出故事片段 |

**验证关卡：** 找来 5 个不认识的真人用户（朋友的朋友即可），给他们发 CLI 脚本（或包装一个非常简陋的 coze bot / 微信机器人），看他们是否愿意用第二次。

### Phase 1: MVP 上线（10 周）

**目标：** 用户可以在手机上使用，获得完整的"说想法 → 看世界线 → 提约束 → 拿片段 → 保存或分享"体验。

| 周 | 内容 | 可交付 |
|----|------|--------|
| 7-8 | PWA 壳搭建，对话界面 V1 + 世界线卡片展示 | 手机上能打开、能打字、能看到生成的卡片 |
| 9-10 | 收敛界面（用户用文本/点选加约束）+ 故事片段展示 + 保存到档案馆 | 端到端 MVP 完整流程 |
| 11-12 | 轻量故事编辑器（Markdown，编辑/标签/串联）+ 发酵池 V1（世界线自动存档） | 用户可以编辑已生成的内容，未选方向自动保留 |
| 13-14 | 表层蒸馏管道（stylometry 统计）+ 用户可上传 .txt 蒸馏 + 用户风格档案 V1 + 行为蒸馏（隐式信号采集） | 系统在 3 次使用后开始"了解"用户偏好 |
| 15-16 | 开放注册 + 基础分享功能（生成卡片/链接）+ 免费版限制生效 | MVP 正式上线对外 |

### Phase 2: 蒸馏与学习（14 周开始，与 Phase 1 部分重叠）

| 周 | 内容 | 可交付 |
|----|------|--------|
| 17-20 | 中层蒸馏管道（LLM 分析情节结构/情感弧线/人物网络） | 用户可以蒸馏中篇以上的作品 |
| 21-22 | 发酵池 2.0（跨会话唤醒 + 多世界线自动融合）+ "你还记得这条吗？"提醒 | 长时间使用后系统能产生"惊喜感" |
| 23-24 | 蒸馏包发布功能 + 市场 V1 + 官方蒸馏包（10 个） | 用户可分享蒸馏包；市场初期供给充足 |

### Phase 3: 商业闭环

| 时间 | 内容 | 触发条件 |
|------|------|---------|
| 月活突破 1000 | 订阅上线（¥39/月 + ¥79/月） | 不早于 Phase 1 完成后 2 个月 |
| 活跃蒸馏包数破 50 | 蒸馏包市场开放定价 + 平台抽成 30% | 市场供需两侧活跃 |
| 月付费用超过 500 人 | 企业版（面向工作室/网文团队，按席收费 ¥199/月/席） | 有 B 端需求信号 |
| 高级阶段 | 深层蒸馏管道开放（需要数据积累） | 积累 10000+ 次用户交互后 |

## 六、商业计划

### 定价体系

| 层级 | 价格 | 核心权益 |
|------|------|---------|
| **免费版** | ¥0 | 每月 30 次发散（每天 1 次）、5 条/次、表层蒸馏 3 次/月、50 个存档、仅文本导出、发酵池 3 条 |
| **专业版** | ¥39/月 或 ¥398/年 | 无限发散、8-12 条/次、全层蒸馏 30 次/月、无限存档、全格式导出、发酵池 20 条 |
| **创作者版** | ¥79/月 或 ¥798/年 | 专业版全部 + 蒸馏包市场发布权 + 可创建 5 个团队空间 + 优先新原子库 + 企业级导出 |
| **企业版** | ¥199/月/席（3 席起） | 团队协作 + 共享风格档案 + 私有原子库 + 定制蒸馏 + SSO |

**蒸馏包市场：**
- 用户定价区：¥5-25/包，平台抽成 30%
- 官方定价区：¥10-25/包，平台 100%
- 免费区：0 元，仅获取声望（积分、榜单排名）

### 成本结构

| 成本项 | 说明 | 预估 |
|-------|------|------|
| LLM API | 每次发散-收敛约 0.5-2 万 token（取决于输出长度） | 若使用 gpt-4o-mini，免费用户单次成本 ~¥0.01-0.03 |
| 服务器（轻量） | PWA 静态托管 + 后端（按需，非实时计算可 serverless） | MVP 阶段每月 ¥200-500 足够 |
| 文件存储 | 用户上传的蒸馏文件 | 初期每月 ¥50-100 |
| 域名 + CDN | 一个域名 + Cloudflare | ≤ ¥200/年 |

**免费用户单次交互成本约 ¥0.02-0.03（用 gpt-4o-mini 或 deepseek-chat）。** 每月 30 次 = ¥0.6-0.9。考虑到部分用户不会用满、大部分行为只是浏览世界线（比完整生成成本低），免费层的实际人均成本低于 ¥0.5/月。

### 收入预估（保守）

| 时间点 | 免费 MAU | 转化率 | 付费用户 | 月收入 |
|--------|---------|--------|---------|-------|
| Phase 1 完成后 2 个月 | 1000 | 3% | 30（专业版） | ¥1,170 |
| Phase 2 完成 | 5000 | 5% | 250（200 专业 + 50 创作） | ¥11,750 |
| 一年后 | 20000 | 5% | 1000（750 专业 + 200 创作 + 50 企业 5 席） | ¥63,050 |
| 加上蒸馏包市场（假设 30% 付费用户每月买 1 个 ¥10 包） | — | — | — | +¥3,000/月 |

**一年后的保守月收入：¥66,050（约 $9,000）。** 不含企业版加购和年付溢价。这不是改变人生的数字，但足够支撑一个 2-3 人小团队持续迭代。

### 获客策略（按优先级）

| 策略 | 成本 | 预期效果 |
|------|------|---------|
| **风格档案社交名片**：用户可视化展示个人叙事 DNA，可分享 | 几乎零成本 | 每个分享 = 一次展示，设计得好可有 5-10% 点击转化 |
| **每日灵感卡片**：系统从公开选集中生成卡片，含二维码 | 每月 ¥100 设计外包（或自己做） | 持续的品牌曝光 |
| **蒸馏包"试读"前 200 字**：购买前即可体验效果 | 零成本 | 降低购买决策门槛 |
| **小红书 / 即刻 内容输出**：发"用 Æsir 做一个 xx 风格的故事"的对比测评 | 时间成本 | 种子用户来源 |
| **邀请机制**：每邀请 1 个好友注册延长 5 次免费发散 | 零成本（边际成本极低） | 免费传播杠杆 |

**不建议前期花的钱：** 应用商店投放、搜索引擎广告、KOL 付费推广。这些应该在产品有 1000 个自然增长用户之后再考虑。

## 七、风险清单

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|---------|
| LLM API 成本过高，免费层亏损 | 中 | 高 | 默认使用低价模型（gpt-4o-mini / deepseek）；发散-收敛管线可切换本地模型（Ollama） |
| 用户发酵池世界线永不被唤醒，变成死数据 | 高 | 中 | 设定过期策略（30 天未被注意的世界线自动归档为"过期"而非删除，用户可手动恢复）；跨会话唤醒加"最近在看什么"的闲聊触发 |
| 蒸馏的版权风险 | 中 | 高 | 只存储元数据（分布、频率、特征向量），不存储原文；用户协议中明确用户对上传内容负责；法务咨询后再开放蒸馏包市场 |
| 手机端体验 vs 桌面深度功能的矛盾 | 中 | 中 | 坚持"轻量编辑器 + 导出到 Pro 工具"的双轨策略，不在移动端堆深度写作功能 |
| 用户"3 次驯化"内流失 | 高 | 高 | 在第三次提供"请告诉我这个系统对你有用吗"的主动对话（非评分表）；如果前两次对话质量够高，第三次的发散即使不完美也有情感连接 |

## 八、一句话总结

> **Æsir 不是一个 AI 写作工具，而是一个随着你的使用不断生长的叙事神经系统——你的每一个模糊想法、每一次选择、每一段蒸馏，都在训练它更准确地触发你作为一个独特创作者的故事直觉。**
