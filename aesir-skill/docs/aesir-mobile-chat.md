# Æsir 手机端对话方案

核心问题：用户在手机上跟谁对话来创作故事？

---

## 三个选项

| 方案 | 用户安装步骤 | 对话体验 | 发散/收敛 | 审计/指纹 | 与 PWA 同步 |
|------|------------|---------|----------|----------|-----------|
| **① Coze Bot** | 打开 Coze app → 搜索安装 Bot | ✅ 原生手机对话 | ✅ LLM 驱动 | ✅ 内置 | ❌ 需手动复制 |
| **② ChatGPT GPTs** | 打开 ChatGPT → 搜索 GPTs | ✅ 原生手机对话 | ✅ LLM 驱动 | ✅ 内置 | ❌ 需手动复制 |
| **③ PWA 内嵌对话** | 打开浏览器 → 添加到主屏幕 | ⚠️ 网页聊天框 | ❌ 无 LLM | ❌ 无 LLM | ✅ 同在 PWA |

**③ PWA 内嵌对话不可行**，因为 PWA 不调用 LLM，没有自己的推理能力。

所以手机端对话只能靠①或②——用户在自己的 agent 平台里完成。

但问题接踵而来：手机端 Coze Bot 生成的片段怎么到 PWA 存档？

---

## 数据流方案

### 当前状态（手动复制）

```
Coze Bot（手机）→ 生成故事 → 用户复制文本 → 打开 PWA → 粘贴 → 存入 localStorage
```

**可行，但不丝滑。** 用户每次都要切换 app、复制、粘贴。

### Phase 1 方案（本地优先）

不需要后端，不需要配置。用户在 Coze 中跟 Bot 对话，Bot 生成故事，用户手动粘贴到 PWA。

PWA 是纯本地模式，数据存 localStorage。跨设备靠导出/导入。

```
流程：Coze 对话 → 复制 → 打开 PWA → 自动检测剪贴板 → 弹出"检测到新内容，要保存吗？"
```

PWA 可以监听 `visibilitychange` 事件，当用户从 Coze 切回 PWA 时，自动读取剪贴板内容，提取故事片段，询问用户是否保存。这样只需要"复制→切回 PWA→点确认"三个动作，不需要手动粘贴。

### Phase 2 方案（云端同步）

配 CloudBase 后，Coze Bot 在生成故事的同时 HTTP POST 到 CloudBase，PWA 每 15 秒自动拉取新内容。

```
流程：Coze 对话 → Bot 自动 POST → PWA 自动拉取 → 用户打开就看到新故事
```

**零操作。**

## 用户完整旅程

### 第 1 次使用

1. 手机应用商店搜索"Coze"并安装
2. 打开 Coze app，搜索"Æsir 故事织机"
3. 安装 Bot，开始对话
4. 说"帮我想个故事"
5. Bot 发散 5 条世界线
6. 用户选一条，Bot 生成故事片段
7. 复制文本
8. 打开 PWA 故事簿（添加到主屏幕后的快捷方式）
9. PWA 自动检测剪贴板 → 弹出"检测到新故事，要保存吗？"
10. 点击保存 → 存入 localStorage → 出现在时间线中

### 之后的每次使用

1. 打开 Coze app → 跟 Bot 对话 → 生成故事
2. 复制 → 切回 PWA → 自动弹出保存提示
3. 点确认 → 完成
