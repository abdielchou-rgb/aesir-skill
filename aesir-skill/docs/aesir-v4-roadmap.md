# 故事沙盒、协作叙事、从单元到作品 — 实施计划

基于对 Coze Bot 能力边界的评估，三个方向在 Coze Bot 平台上能做多少、需要什么条件。

---

## 方向一：故事沙盒

**核心理念：** 用户反复做"发散→选择→收敛→调整"的循环，每次调整后的结果保留在画布中，随时可以退回到之前的某个决策点重新走。

### 能做的部分（在 Coze Bot 内）

| 功能 | 实现方式 | 难度 |
|------|---------|------|
| 画布中所有单元的可视化 | system prompt 中维护一个"画布摘要"，用户说"画布"时展示 | 低 |
| 退回到之前的分歧点 | 在画布摘要中记录每个单元的分歧点，用户说"回到第X单元的分歧点A"时重设上下文 | 中 |
| 多个单元的线索交叉 | 画布摘要中记录所有未闭合线索，用户说"A单元的线索X和B单元可以结合吗"时在prompt中做匹配 | 中 |

**示例对话：**

```
用户：画布
Bot：你目前有 3 个单元：
  · 单元A "地铁上的女孩"（悬疑）- 未闭合线索：照片里的男人是谁
  · 单元B "老房子"（情感）- 未闭合线索：钥匙对应的锁在哪里
  · 单元C "咖啡馆重逢"（爱情）- 未闭合线索：她为什么离开这个城市
  
  三个单元的线索中有重叠关键词"离开/照片/男人"，要尝试连接吗？
```

### 不能做的部分（超出 Coze Bot 能力）

| 功能 | 为什么不能做 | 需要什么 |
|------|------------|---------|
| 倒退到任意决策点并重新生成 | Coze 工作流没有"状态回溯"的能力。一旦收敛完成，之前的输入就丢失了 | 需要本地状态管理（PWA + localStorage 或 MCP Server） |
| 可视化节点图 | Coze 只支持文本对话，不支持图形界面 | 需要 PWA 前端 |

### 实施建议

沙盒的"轻量版"可以在 Coze Bot 中做：
- system prompt 中维护画布摘要
- 用户说"画布"时展示所有单元和未闭合线索
- 用户说"交叉线索"时做关键词匹配
- 用户说"回到XX"时根据画布中的记录重新设定上下文

真实沙盒（可视化倒退+节点图）需要 PWA 配合，Coze Bot 做不到。

## 方向二：协作叙事

**核心理念：** 多个用户的画布可以共享。A 延展了 B 的一个未闭合线索，系统自动标注"这条线索由 B 开启、由 A 延展"。

### 在 Coze Bot 中的可行性：❌ 不能做

| 限制 | 原因 |
|------|------|
| Coze Bot 是单用户对话 | 多个用户之间没有共享的存储空间 |
| 没有用户身份系统 | Coze 不提供跨会话的用户标识 |
| 不能写入共享数据库 | Coze 的 HTTP 调用可以 POST 到 CloudBase 等后端，但涉及用户认证和权限 |

### 必要条件

- 一个后端服务（CloudBase 或自建）
- 用户认证（至少是简单的 shared ID）
- 共享数据库（记录每个用户的画布、开放线索、延展记录）

### 实施建议

**现在不做。** 等单用户画布稳定、用户量上来了再启动。协作叙事是"锦上添花"的功能，核心功能（单用户从模糊想法到完整故事）才是基础。

---

## 方向三：从故事单元到完整作品

**核心理念：** 画布中积累的多个单元可以自动检测"这些单元是否能拼成一个更大的故事结构"——基于未闭合线索的交叉程度和角色的重叠度。

### 能做的部分（在 Coze Bot 内）

| 功能 | 实现方式 | 难度 |
|------|---------|------|
| 单元自动合并 | 用户说"把单元A和单元B合并"时，在prompt中传入两个单元的内容，要求LLM融合 | 低 |
| 线索匹配检测 | 基于关键词重叠——两个单元都出现了"照片"这个词，可以建议合并 | 中 |
| 按时间线排列 | 用户说"把我所有的单元按故事先后排序"时，LLM分析每个单元的时间线索 | 低 |

**示例对话：**

```
用户：合并单元A和单元B
Bot：两个单元的未闭合线索中都有"照片"和"失踪的父亲"。
A 的调性是悬疑暗流，B 是温暖治愈。
合并后有两种可能：
1. 以悬疑为主线，情感线为副线（推荐——更符合两个单元的走向）
2. 以情感为主线，悬疑为副线

选哪个？或者我先写一段看看效果？
```

### 不能做的部分

| 功能 | 为什么不能做 |
|------|------------|
| 自动检测所有单元之间的潜在连接 | 需要批量文本分析，Coze 的工作流一次只能处理一个输入 |
| 生成完整的长篇结构（50+章节） | Coze 的单次 token 限制不足以支撑完整长篇 |

### 实施建议

合并功能可以在 Coze 中做一个"merge"工作流：

```
merge 工作流：
  输入：单元A的内容 + 单元B的内容 + 合并方向
  LLM 节点：基于两个单元的线索和角色，生成合并后的新故事单元
  Code 节点：解析输出，更新未闭合线索列表
```

---

## 优先级和实施路径

| 方向 | 在 Coze 中能做 | 推荐优先级 |
|------|---------------|-----------|
| 从单元到作品（单元合并） | ✅ 可以做 | **P0** — 已有画布中积累多个单元后，合并是自然需求 |
| 故事沙盒（画布+回溯） | ⚠️ 轻量版可以做 | **P1** — 画布可视化+线索交叉可以在 system prompt 中改进 |
| 协作叙事 | ❌ 不能做 | **P2** — 需要后端基础设施 |

### 近期可做的

**merge 工作流：** 在 Coze Bot 中新增一个工作流，输入两个单元的内容和合并方向，输出合并后的新单元。这是三个方向中唯一可以在 Coze 中完整实现的。
