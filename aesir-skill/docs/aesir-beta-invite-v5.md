# Æsir 故事织机 — 内测邀请

> 一个拖进 Claude 就能用的故事伙伴。
> 不卖课、不收费、不替你写——只是帮你在动笔之前多看看可能性。

---

## 我们为什么要做这个

每个人心里都藏着几个故事。地铁上看到的陌生人、睡前冒出来的"如果当时……"、一种说不清楚但挥之不去的情绪。大多数故事到这里就停了——不是因为写不出来，是因为**没人帮你把那团模糊的感觉变成一个"可以开始"的方向**。

市面上的 AI 写作工具分两种。一种是给你一段一段生成文本的代写机器——投到平台 AI 检测一查就拒，写出十万字人设全崩，读者看完说"没人味"。另一种是卖课的说"三天学会用 AI 写爆款网文"——模板套框架、框架套爽点、爽点塞进模型，出来的东西自己都不想读第二遍。

我们想做的不一样。

我们想做一个**编剧室里的编剧搭子**。你跟它说一个念头，它帮你发散几条方向，你选一条，它帮你收敛成一个完整的故事单元。每个单元有开头有结尾，但会留几条未闭合的线索——你想继续的时候可以沿着线索往下走。整个过程你不是在做选择题——你是导演，它是编剧室。

它的知识底座不是从哪篇文章里抄的——是从麦基的《故事》、Truby 的对手设计、Cron 的神经科学叙事、网文行业十几年的爽点排布经验里长出来的。它**不替你写**不是因为不能写——是因为它知道 AI 写的完读率只有人工的 61%、情感共鸣度只有 55%、十万字后人设崩塌率超过 60%。它存在的意义是在你写之前、写之中、写之后帮你看到盲区。

**我们最终想实现的目的很简单：让每个想讲故事的人，都有一个不会嫌麻烦、不会要你付钱、不会替你写、但会在每一个岔路口帮你多看一眼的搭档。**

## 它的设计逻辑

它的底层只有一件事：**发散和收敛的交替循环**。这不是从 AI 写作工具借鉴的——是创意心理学研究了五十年的结论。Campbell 的 BVSR 理论（盲变与选择保留）说真正的创意不是从一堆选项里挑最好的，而是先**大量盲目变体**——把你从来没想过的方向全摊到桌上——再**选择性保留**——你作为创作者凭直觉判断哪个方向有感觉。

具体怎么做呢？你说了一个模糊想法。系统不是直接吐一段文字——它先用 5 种不同的创意人格各看一遍你的想法。浪漫主义者看到的是一个关于连接的故事，黑色幽默者看到的是荒诞，反乌托邦者看到的是规则与反抗，理性解谜者看到的是需要推开的门，神话讲述者看到的是古老的重复。5 条世界线——每条带着一个具体的情感锚点——不是"她很悲伤"，是"她接过纸巾时手指轻轻碰了碰对方的手，然后立刻缩回去"。

你选一条。系统收敛成 800-1200 字的完整故事单元。所谓"完整"不是字数够了——是走完了 SG5 结构的一整圈：煽动事件 → 转折点 → 危机时刻 → 高潮 → 解决。单元结束了，但还留了 2+ 条未闭合的线索——你可以沿着任何一条继续往下走，也可以把这个单元计入画布，开始下一个想法。

等画布里攒了几个单元，系统会自动检测交叉点——A 单元的线索 X 和 B 单元的线索 Y 有没有关键词重叠。有的话提醒你："这两个单元可以合并，要不要试试？"

审计是另一层。每个单元生成后可以跑 15 组启发式门禁——不分析文笔，只分析结构：鸿沟密度够不够、黄金三章有没有冲突、章尾有没有钩子、对话是不是全完整句子（AI 腔的致命信号）、情感是不是被解释了而不是被展示了。

你的角色始终是同一个：导演。系统的角色始终是同一个：编剧室里那个不怕麻烦的搭档。

## 受谁的影响

这个工具的知识底座吸收了 30+ 个理论来源，但它们不是装饰——每个都在系统行为里有一个具体的落地位置。

**叙事结构层**。麦基的 Story Grid 5 诫命决定了收敛单元的结构完整性——每个单元必须走完煽动→转折→危机→高潮→解决一整圈。Snyder 的 Save the Cat 15 节拍、Pixar 的 Story Spine、Harmon 的故事圆分别在不同的故事类型里被激活。MICE Quotient 的四种故事类型和嵌套规则决定了发散时怎么选择视角。

**角色和冲突层**。Truby 的对手设计理论——对手不是"坏人"，是持有相反价值观的人——在画布展示时体现为"这个角色的价值观在这个单元里发生了什么变化"。McKee 的三层冲突（自我、人际、社会）是审计门禁的基准——一个故事如果只有外部冲突没有内心矛盾，门禁会亮 WARN。

**微观写实层**。Cron 的神经科学叙事——大脑通过感官细节建构情感——决定了收敛时的文本质感：用动作和反应展示情感，不解释。Maass 的微张力——每页必须有一个未解释的事情——在审计门禁中体现为 MTS-01。Swain 的 MRU 反应链——刺激→情感→思考→动作→言语不可跳跃不可逆序——影响的是修改建议的生成逻辑。

**网文实战层**。12 种爽点原子不是从哪篇文章抄的概念——是爬了一百多本起点榜单书之后归纳出来的。网文读者不靠情节记故事，靠爽点——信息差碾压、降维打击、认知颠覆、以牙还牙、绝境反转——这些不是理论，是读者的生理反应。类型契约表是这层的核心——悬疑前 3 章必须建异常，仙侠第 10 章必须首次大突破，言情 CP 感前 3 章必须建立——不是在教怎么写，是在告诉你读者**什么时候会弃书**。

**学术前沿**。2026 年 CHI/AAAI 的最新研究也在影响方向——NarrativeLoom 的多智能体创意人格分化直接改变了发散机制（5 条世界线来自不同人格视角而非同一概率分布的不同采样），DiriGent 的动态信念系统正在被探索应用到画布中的角色弧线追踪。

## 你怎么用

**零配置。** 没有 API Key 要配，没有 Python 要装，没有注册要填。

把 `aesir-skill.md` 这个文件拖进 Claude Desktop 的 Project Knowledge 区。或者把内容粘贴到 Coze/豆包/ChatGPT 的 System Prompt 里。然后说"帮我想个故事"。

前 3 次使用，它会先问你 1-2 个问题理解你真正想要什么。3 次之后跳过驯化，直接发散。

你说了想法后，它会用 5 种不同的创意人格给你 5 条世界线——每条有一句精确的前提句加一个具体的情感锚点。你选一条。它收敛成完整故事单元。你说"继续"→沿线索续写。你说"画布"→展示所有单元。你说"审计"→15 组门禁检查。

想用更深的？配合文鉴 WenJian（`pip install wenjian && wenjian mcp-server`），131 道门禁 + 15 维风格指纹 + 爆款对标——全本地，零 API Key。

## 我们需要你测什么

内测阶段不需要你写测评报告。你正常用，然后告诉我——不是告诉我"好不好用"，是告诉我以下五件事里的任何一件：

1. 发散出来的 5 条世界线，你有没有一两条觉得"换了个标签但骨子里跟另一条一样"？
2. 收敛出来的故事单元，读完后的第一反应是"还行，想继续往下看"还是"就这？"
3. 你选了某个类型（比如悬疑），系统有没有前 3 章帮你建了异常？如果你选了言情，CP 感前 3 章有没有建立？
4. 审计给的结果和你自己的判断一致吗？有没有你觉得写得好的段落被门禁标了 WARN，或者你觉得有问题的段落门禁没抓到？
5. 生成的内容里有没有让你觉得"这就是 AI 写的"的地方？你改完后有没有信心过平台检测？

如果某个地方让你觉得"不对"，直接改 `aesir-skill.md`。它是纯 Markdown，改里面的 prompt 就能调行为。改了之后告诉我效果，好的话我会合并到主版本。
