# 发散收敛时要不要提供可能性大纲？

核心问题：当前 converge 工作流直接生成 800-1200 字的完整故事单元。但用户的模糊想法可能只需要一个骨架，而不是直接看到成品。要不要在"发散世界线"和"收敛故事单元"之间，增加一个"可能性大纲"的中间层？

---

## 当前流程

```
模糊想法 → diverge(5条世界线) → 用户选一条 → converge(800-1200字完整故事单元)
```

## 建议的新流程

```
模糊想法 → diverge(5条世界线) → 用户选一条 
  → ① 大纲层(可选): 2-3个可能性大纲,每个大纲是5-8个节拍点的骨架
  → ② 收敛层: 用户选中一个大纲方向后,收敛为完整故事单元
```

---

## 学术界和实践界的依据

### 支持加中间层的

| 来源 | 为什么支持 |
|------|----------|
| **Snyder Save the Cat 15节拍** | 节拍本身就是一个"可能性大纲"。每个节拍是一个"位置"而非"内容"——"中点"可以是假胜利也可以是假失败,这本身就是一种发散 |
| **Story Grid 5诫命** | 煽动/转折/危机/高潮/解决——这5个点构成了一个完整叙事单元的骨架。用户可以先看到骨架,再决定要不要填血肉 |
| **Yorke 5幕辩证** | 正/反/合/超越/余韵——每个层次都容纳多种具体的实现方式 |
| **Campbell 17阶段** | 不是每个故事都需要走完17步,但知道"你现在在哪个阶段"能帮助用户判断故事是否健康 |
| **网文行业** | 网络作家不是直接写全文的,是先写"细纲"——每一章的核心事件和情绪落点。这就是一种节拍大纲 |
| **Genette 故事/话语区分** | "可能性大纲"属于故事层(发生了什么),"完整故事"属于话语层(怎么讲述)。把这两层分开,用户可以有更多选择 |

### 不支持加中间层的

| 来源 | 为什么反对 |
|------|----------|
| **Cron 神经叙事学** | 读者(和用户)需要情感卷入才能保持注意力——大纲太抽象,没有感官细节,无法激活读者的神经模拟。如果用户看了大纲觉得"没感觉",可能就放弃了,其实完整故事可能会打动ta |
| **Pixar 22条规则** | "先落纸再改"——不要在大纲阶段过度优化,先写出来再说。再加一个选择层可能让用户陷入"选哪个方向"的循环 |
| **Maass 微张力** | 微张力是在细节层面产生的,不是在大纲层面。大纲看起来平平无奇的方向,填充细节后可能非常有张力 |
| **金牌编剧共识** | Sorkin、Mazin等人都说过同样的话——结构是事后发现的,不是事前设计的。大纲会让你过早锁定一个方向,错过了故事自己生长的可能 |

### 学术界的争议

**Dramatica 叙事意图模型**的核心立场是：**故事的质量取决于叙事意图的精确度,而不是场景的丰富程度**。从这个角度看,在收敛之前先做"意图对齐"(用户说"我其实想要的是XX,不是XX")比"大纲发散"更有价值。

**而 Geneplore 模型**(Finke, Ward & Smith)认为创意过程是"生成"和"探索"的交替循环。在这个框架下,大纲是一个"前创意结构"(pre-inventive structure),它需要在"探索"阶段被检验和变形。所以大纲应该轻、可变、不建议用户锁定。

---

## 我的判断

**加一个轻量级的大纲层,但不作为必选项,而是作为用户可选的能力。**

具体做法：

| 用户说 | 系统做 |
|--------|--------|
| "帮我想个故事" | 发䋎+收敛(直接出故事单元,当前流程) |
| "帮我想个故事,先给我看看大纲" | 发䋎+大纲(节拍骨架)+用户选方向+收敛 |
| "这个方向有意思,大纲会是什么样?" | 在已有世界线上生成节拍大纲 |

这样,大纲层是一条可选路径,不是必经之路。新手用户可以直接收敛出完整故事,有经验的用户可以选择先看骨架再决定填充方向。

---

## 补充：一个更重要的改进

你说的"可能性大纲"让我想到一个更重要的事——当前发散出来的5条世界线,每条只有一句话的前提句和一个150字的钩子。**用户很难只凭这些信息做选择。**

一个更好的设计可能是：发散时不只是给出前提句,而是给出一个"叙事意图卡片"：

| 字段 | 示例 |
|------|------|
| 前提句 | 当一个试图忘记过去的人在陌生城市遇到一个知道她全部秘密的人 |
| 主角欲望/需求 | 她想要重新开始,需要面对过去 |
| 冲突层次 | 人际层(她vs知道秘密的人) + 自我层(她vs自己的记忆) |
| 情感弧线 | 从逃避到面对(冷峻写实调性) |
| 核心悖论 | 重新开始的唯一方式是先完成未完成的事 |
| **可能性方向A** | 对方是善意的人,帮她面对过去(温暖治愈) |
| **可能性方向B** | 对方利用这个秘密控制她(悬疑暗流) |
| **可能性方向C** | 这个秘密其实不是关于她的——是她忘记的另一个人的事(认知颠覆) |

用户看到这样的卡片,更能判断"这个方向我有没有感觉",而不是看一句话的前提句猜。

---

## 总结

| 角度 | 结论 |
|------|------|
| 是否加中间大纲层 | 加,但作为可选能力,不是必经之路 |
| 大纲的形式 | 5-8个节拍点的骨架,而不是详细段落 |
| 更重要的是 | 扩展发散的输出内容——从"前提句+钩子"扩展为"叙事意图卡片+多个可能性方向" |

要不要我把这个"叙事意图卡片+可能性方向"的设计加到当前的 Coze Bot 中？改动在 diverge 工作流的 LLM prompt 里加一个输出格式要求就行。
