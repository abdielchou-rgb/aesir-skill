Æsir 故事织机 v5.0 — 内测邀请

把模糊的念头变成完整的故事单元，像织布一样，一线一线织出你的故事。


我们为什么要做这个

每个人心里都藏着几个故事。地铁上看到的陌生人、睡前冒出来的"如果当时……"、一种说不清楚但挥之不去的情绪。大多数故事到这里就停了——不是因为写不出来，是因为没人帮你把那团模糊的感觉变成一个"可以开始"的方向。

市面上的 AI 写作工具分两种。一种是代写机器——生成完就没了，投到平台 AI 检测一查就拒，写出十万字人设全崩。另一种是模板套壳——框架套爽点、爽点塞模型，出来的东西自己都不想读第二遍。

我们想做的不是一个生成器。我们想做一个不会嫌麻烦的创作搭档。你跟它说一个念头，它帮你发散几个方向，你选一条，它帮你收敛成一个有开头有结尾的故事单元。你不是在做选择题——你是导演，它是编剧室。

它不替你写，不是因为不能写，而是因为 AI 生成的文本完读率只有人工的 61%，情感共鸣度只有 55%。它存在的意义是在你写之前、写之中、写之后帮你看到盲区——然后你自己写。


它的设计逻辑

底层只有一件事：发散和收敛的交替循环。这不是从 AI 工具借鉴的——是创意心理学研究了五十年的结论。真正的创意不是从一堆选项里挑最好的，而是先大量盲目变体，再凭直觉选择性保留。

你说了一个模糊想法。它用 5 种不同的创意人格各看一遍你的想法——浪漫主义者看到连接，黑色幽默者看到荒诞，反乌托邦者看到规则与反抗。5 条世界线，每条带着一个具体的情感锚点。你选一条。它收敛成 800 到 1200 字的完整故事单元——走完 SG5 结构的一整圈：煽动、转折、危机、高潮、解决。单元结束了，但留了至少两条未闭合的线索。你可以沿着任何一条继续，也可以计入画布，开始下一个想法。

等画布里攒了几个单元，系统会自动检测交叉点，提醒你哪些单元可以合并。


它受了谁的影响

它的知识底座吸收了 30 多个理论来源，每个都在系统行为里有一个具体的落地位置：

叙事结构层——Story Grid 的 5 诫命决定了收敛单元的完整性，Save the Cat 的 15 节拍、Pixar 的 Story Spine 在不同类型里被激活。

角色和冲突层——Truby 的对手设计理论决定了画布里角色弧线的展示方式，McKee 的三层冲突是审计门禁的基准。

微观写实层——Cron 的神经科学叙事决定了文本"用动作展示、不解释情感"的质感，Maass 的微张力内置在审计中。

网文实战层——12 种爽点原子来自一百多本起点榜单书的归纳，类型契约表告诉你每种类型的读者在什么时候会弃书。

学术界前沿——2026 年 CHI 和 AAAI 的最新研究（NarrativeLoom 多智能体创意人格分化、DiriGent 动态信念系统）直接影响发散和画布机制。


你怎么用

零配置。没有 API Key 要配，没有 Python 要装，没有注册要填。

1. 下载附件 aesir-skill.md，保存到你能找到的地方。
2. 拖入任意 AI Agent 的附件、知识区或 System Prompt 中。主流平台都支持。
3. 说"帮我想个故事"，或者随便输入一个念头。

前三次使用，它会先问你一两个问题。三次之后跳过，直接发散。

想配合深度审计？安装文鉴（pip install wenjian），131 道门禁、15 维风格指纹、爆款对标——全本地，零 API Key。


帮我们测什么

不需要写测评报告。正常用，然后把所有让你觉得"不对"的地方告诉我：

- 发散出来的 5 条世界线有没有"换了个标签但骨子里一样"的？
- 收敛出来的故事单元，读完想继续看，还是"就这？"
- 选了某个类型（比如悬疑），前几章有没有按类型契约来？
- 审计结果和你自己的判断一致吗？有没有误报或漏报？
- 生成的内容有没有"AI 腔"？你改完后有没有信心过平台检测？
- 任何让你想吐槽的地方——语气、节奏、逻辑、内容——随便说。
